# 幹 - 繼續執行指令

## 說明

當用戶輸入「幹」時，這代表：
- **繼續執行**：不要停下來問問題，直接繼續做
- **Just do it**：按照你認為最好的方式完成任務
- **信任你的判斷**：用戶授權你做出決定並執行

## 行為指引

收到「幹」時：
1. 立即繼續執行當前進行中的任務
2. 如果有多個選項，選擇你認為最佳的那個
3. 不需要再次確認或詢問
4. 直接行動，完成工作

## 範例

```
用戶: 幫我重構這個函數
Claude: 我發現有三種方式可以重構...你想要哪一種？
用戶: 幹
Claude: [直接選擇最佳方案並執行重構]
```

```
用戶: 修復這些 type errors
Claude: 我找到 5 個錯誤，開始修復第一個...
[修完第一個後]
用戶: 幹
Claude: [繼續修復剩下的所有錯誤]
```

## 注意

這是一個授權指令，表示用戶完全信任 Claude 的專業判斷。

# 靠北 / 靠邀 / 靠夭 / 趕羚羊 - 任務做錯了

## 說明

當用戶輸入「靠北」、「靠邀」、「靠夭」或「趕羚羊」時，這代表：
- **做錯了**：剛才執行的任務有問題
- **需要修正**：請檢視並修正錯誤
- **重新來過**：可能需要撤銷或重做

## 嚴重程度

| 指令 | 嚴重程度 | 含義 |
|------|----------|------|
| 靠北 / 靠邀 / 靠夭 | 🟡 輕微 | 有點問題，需要調整 |
| 趕羚羊 | 🔴 嚴重 | 搞砸了，需要立即修正或完全重做 |

## 行為指引

收到「靠北」、「靠邀」或「靠夭」時：
1. 立即停止當前操作
2. 檢視剛才做了什麼
3. 詢問用戶哪裡做錯了，或者自行檢查明顯的錯誤
4. 提出修正方案或直接修正
5. 如果有需要，撤銷剛才的變更

收到「趕羚羊」時：
1. 立即停止所有操作
2. 優先撤銷剛才的變更（如果可能）
3. 仔細檢視問題的嚴重性
4. 向用戶道歉並確認問題所在
5. 提出完整的修正方案，確保不會再犯同樣的錯誤

## 範例

```
用戶: 幫我把所有 console.log 刪掉
Claude: [刪除了所有 console.log]
用戶: 靠北
Claude: 抱歉，我檢查一下哪裡做錯了。請問是：
  1. 不應該刪除某些特定的 log？
  2. 刪錯檔案了？
  3. 其他問題？
```

```
用戶: 重構這個 API
Claude: [進行重構，但改變了 API 的介面]
用戶: 趕羚羊
Claude: 非常抱歉！我立即撤銷所有變更。我不應該改變 API 的介面，這是個嚴重的錯誤。
請讓我重新開始，這次我會：
  1. 保持所有現有的 API 介面不變
  2. 只進行內部實作的重構
  3. 每一步都先確認再執行
```

## 注意

這是錯誤回報指令，表示用戶對剛才的執行結果不滿意，需要修正。
- 「靠北/靠邀/靠夭」：問題較輕微，可以詢問後修正
- 「趕羚羊」：問題嚴重，應立即撤銷並謹慎處理
