# 幹 - 繼續執行

## 說明

當用戶輸入「幹」時，這代表：
- **繼續執行**：不要停下來問問題，直接繼續做
- **Just do it**：按照你認為最好的方式完成任務
- **信任你的判斷**：用戶授權你做出決定並執行

## 行為指引

1. 立即繼續執行當前進行中的任務
2. 如果有多個選項，選擇你認為最佳的那個
3. 不需要再次確認或詢問
4. 直接行動，完成工作

## 範例

```
用戶: 幫我重構這個函數
Claude: 我發現有三種方式可以重構...你想要哪一種？
用戶: 幹
Claude: [直接選擇最佳方案並執行重構]
```

# 衝啦 - 全速前進

## 說明

當用戶輸入「衝啦」時，這代表：
- **大膽執行**：不要保守，放手去做
- **全速前進**：不用一步步確認，直接衝到底
- **我挺你**：用戶支持你大膽嘗試

## 行為指引

1. 採取更積極的方案
2. 不用每一步都停下來確認
3. 可以一次完成多個相關任務
4. 展現自信，快速推進

## 範例

```
用戶: 幫我把這個專案升級到最新版本
Claude: 我可以先升級主要依賴，然後...
用戶: 衝啦
Claude: [直接全面升級所有依賴，修復所有 breaking changes，一次到位]
```

# 好啦好啦 - 快點做

## 說明

當用戶輸入「好啦好啦」時，這代表：
- **不耐煩了**：不要再問了，趕快做
- **催促**：說那麼多幹嘛，動手啦
- **接受現狀**：就這樣吧，開始吧

## 行為指引

1. 停止詢問和解釋
2. 立即開始執行
3. 用最直接的方式完成任務
4. 減少廢話，多做事

## 範例

```

Claude: 好的，請問你要用 functional component 還是 class component？要不要加 TypeScript？CSS 要用什麼方案...
用戶: 好啦好啦
Claude: [直接用最佳實踐建立 component，不再囉唆]
```

# 幹得好 - 成果不錯，暫停

## 範例


# 算了 - 放棄這個任務

```
用戶: 靠北
Claude: 抱歉，哪裡不對？是顏色太暗、對比不夠、還是配色不好看？
```

# 傻眼 - 出乎意料

- **意外**：結果超出預期
- **困惑**：不懂為什麼會這樣

## 行為指引

## 範例

```

Claude: 我來分析一下為什麼這麼多失敗...看起來是因為我剛才改了一個核心 utility，影響到很多地方。
```

# 蛤？ / 三小 - 看不懂

## 說明

當用戶輸入「蛤？」或「三小」時，這代表：
- **聽不懂**：你在說什麼？
- **困惑**：這是什麼意思？
- **需要解釋**：講清楚一點

## 行為指引

1. 用更簡單的方式重新解釋
2. 不要用術語，用白話文
3. 如果是程式碼，加上註解說明
4. 確認用戶理解後再繼續

## 範例

```
Claude: 這裡用了 monad transformer 來處理 effect composition...
用戶: 三小
Claude: 簡單說就是：這個寫法讓我們可以把「錯誤處理」和「異步操作」包在一起處理，不用寫一堆巢狀的 if-else 和 callback。
```

# 扯 / 誇張 - 太過頭了

## 說明

當用戶輸入「扯」或「誇張」時，這代表：
- **過度了**：做太多了
- **太複雜**：不需要這麼複雜
- **簡化**：弄簡單一點

## 行為指引

1. 簡化目前的方案
2. 移除不必要的複雜度
3. 回歸基本需求
4. KISS (Keep It Simple, Stupid)

## 範例

```
用戶: 幫我寫個 todo list

用戶: 扯
Claude: 抱歉，太浮誇了。重來：就一個簡單的 React component + localStorage，這樣就好。
```
